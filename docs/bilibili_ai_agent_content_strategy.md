# B站 AI Agent 内容创作策略

> 2026-05-05 基于 CDP 协议实时调研 B站数据，结合本项目能力矩阵分析

## 一、背景

当前项目是一套日本 Yahoo 新闻自动化流水线：
- CDP 协议操控 Chrome 抓取新闻
- LiteLLM 调用 deepseek-v4-pro 做翻译 + 内容生成
- 图集/YouTube 视频自动下载 + 字幕烧录
- 自动发布到小红书

本项目本身就是一个完整的 AI Agent 实战案例。

## 三、内容方向

### 系列一：AI Agent 实战 — 日本新闻自动化流水线

| 期数 | 内容 | 时长 | 素材来源 |
|------|------|------|---------|
| EP1 | 如何用 CDP 协议操控浏览器自动抓数据 | 10-15min | `yahoo_news_auto.py` → `fetch_news_via_cdp` |
| EP2 | LiteLLM 多模型切换 — deepseek-v4-pro 做翻译 | 10min | `yahoo_common.py` → `call_litellm` |
| EP3 | 自动生成小红书内容 — Prompt 工程实战 | 12min | `generate_content_and_comment` |
| EP4 | 图集自动下载 + 去重 + Cloudinary 上传 | 10min | `gallery_fetch.py` → `process_page` |
| EP5 | YouTube 视频下载 + 字幕烧录全自动化 | 12min | `gallery_fetch.py` → `_download_youtube_video` |
| EP6 | 全流程串联 — 从零到发布只需一条命令 | 8min | `yahoo_news_publish.py` → `main` |

## 四、竞争壁垒

| 其他 AI 教程 UP | 本项目优势 |

壁垒不在 AI 知识深度，在于：**程序员 + 日语能力 + 已有自动化流水线** 的三重组合。

## 五、制作流程

```
1. 写逐字稿 → AI 润色（复用 call_litellm）
2. 录屏开发过程（不需要露脸，录 IDE + 终端即可）
3. 剪映自动生成字幕 → 复用 yt-dlp 脚本逻辑
4. 发布到 B站 → 可写 bilibili_publish.py 自动化
```
